# 🏥 آزادی طب (Azadi-Teb)

**سامانه پذیرش و مدیریت درمانگاه — نسخه ۲.۰.۰**

> **نسخهٔ ۲: تحولِ معماری.** لایهٔ نمایش با **Avalonia UI (.NET 9) + MVVM**
> بازنویسی شد (ظاهرِ زیبا، Responsive، RTL، Dark/Light) و منطقِ کسب‌وکار در
> **هستهٔ C++20** حفظ و به‌صورتِ سرویسِ **REST** جدا شد. برنامه دیگر هنگ یا
> کرش نمی‌کند. جزئیات کامل: [`docs/ARCHITECTURE_V2.md`](docs/ARCHITECTURE_V2.md).
>
> - لایهٔ نمایش: `app/AzadiTeb.UI/` (Avalonia، MVVM)
> - هستهٔ C++ REST: `backend/`
> - خروجی نهایی: `build/AzadiTeb.exe` (single-file، self-contained، win-x64)
> - بیلد: `./build.sh` (فقط UI) یا `./build.sh core` (به‌همراه هستهٔ C++)

نسخهٔ قدیمِ Win32 (کد `src/*.cpp`) به‌عنوان مرجع در مخزن باقی مانده است.

## 📦 دانلود و اجرا

فایل اجرایی آماده: [`build/AzadiTeb.exe`](build/AzadiTeb.exe)

- یک فایل EXE واحد — **هم روی ویندوز 32 بیتی و هم 64 بیتی** اجرا می‌شود.
- سازگار با ویندوز **7 / 8 / 8.1 / 10 / 11** و جدیدتر.
- نیاز به نصب ندارد؛ کنار EXE پوشه‌های `data/` و `logs/` خودکار ساخته می‌شوند.
- روی سخت‌افزار ضعیف (RAM 2GB / CPU دو هسته‌ای) هم روان کار می‌کند.

### شروع سریع
1. `AzadiTeb.exe` را اجرا کنید (تمام‌صفحه باز می‌شود).
2. در صفحه اصلی `Ctrl + P + N` را نگه دارید → پنل ادمین → ورود با `prf` / `prf123`.
3. کاربر بسازید (مثلاً بخش: «دندانپزشکی»، نوع: پذیرش).
4. خروج (دکمه ✕ بالا راست) → «پذیرش درمانگاه» → ورود با کاربر ساخته‌شده → انتخاب شیفت → ثبت پذیرش.

## 🛠️ بیلد از سورس

```bash
# روی لینوکس (Debian/Ubuntu):
sudo apt install g++-mingw-w64-i686
./build.sh        # خروجی: build/AzadiTeb.exe
```

روی ویندوز با MSYS2/MinGW32 هم همان فرمان‌های داخل `build.sh` قابل اجراست.

## 📂 ساختار مخزن

```
├── src/             ← سورس C++ (Win32 خالص)
├── fonts/           ← فونت Vazirmatn (داخل EXE تعبیه می‌شود)
├── build/           ← خروجی EXE نهایی
├── docs/            ← مستندات کامل (PROJECT_GUIDE / CHANGELOG / PROMPT)
├── update/          ← فایل version.txt برای آپدیت از راه دور
└── build.sh         ← اسکریپت بیلد
```

## 🔄 آپدیت از راه دور — راه‌اندازی سرور

فایل `update/version.txt` در این مخزن (یا هر سرور HTTP دیگر):
```
1.0.1
https://example.com/path/AzadiTeb.exe
```
(خط اول = آخرین نسخه، خط دوم = لینک دانلود EXE جدید.)
آدرس پیش‌فرض، raw گیت‌هاب همین مخزن است؛ برای تغییر، در `data/settings.ini` کلید `update_url` را تنظیم کنید.
